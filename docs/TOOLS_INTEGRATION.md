# External AI Tools — Integration Guide

> **Purpose**: how Captain / Weekly Ops / Specialists dispatch tasks to external AI tools + receive results.
> **Principle**: every external tool is a **sub-agent**, not a parallel Claude chat. Results flow back to the orchestrating Claude chat via copy-paste or file.
> **Last updated**: 2026-04-20

---

## 🎯 Integration model

```
┌──────────────────┐         ┌─────────────────────┐
│ Claude Code chat │────┬───▶│ External AI tool     │
│ (Captain/Ops/    │    │    │ (Perplexity/NBLM/etc)│
│  Specialist)     │◀───┼────│                      │
└──────────────────┘    │    └─────────────────────┘
                        │
                        │ Handoff prompt
                        │ (generated by Claude,
                        │  executed by user)
                        │
                        ▼
                  Result comes back via:
                  - user copy-paste into Claude
                  - saved file in docs/research/
                  - quote in session scratchpad
```

**No tool-to-tool direct API calls** — keeps architecture simple, costs predictable, user always in loop.

## 🔌 Tool 1: Perplexity Pro (€20/mo)

**When to use:**
- Real-time web research requiring citations (regulations, market data, competitor moves)
- Fresh information (news, API docs, product announcements) that would be stale in Claude's training
- Due diligence on startup programs, vendors, legal precedents
- Romanian/European regulatory updates

**NOT for:**
- Code generation (use Claude Code)
- Document synthesis of files you already have (use NotebookLM)

### User setup (one-time)
1. Account at [perplexity.ai](https://www.perplexity.ai) with Pro subscription (€20/mo)
2. Enable "Pro Search" and "Spaces" features
3. Create a "ModulCA" space to persist research threads (optional but recommended)

### Handoff prompt format (Claude → user → Perplexity)

When Claude needs Perplexity, it outputs a block like this:

```
▶ PERPLEXITY HANDOFF
Mode: Pro Search (cite sources)
Scope: web (or: academic / writing)
Query: [exact query to paste]
Follow-ups if needed: [1-3 related questions]
Expected back: [summary + top 3 citations]
```

### User action
1. Open Perplexity Pro in browser
2. Paste the query from the handoff block
3. Wait for answer
4. Copy the answer (including citation links) back to Claude chat
5. Optional: save to `docs/research/YYYY-MM-DD-<topic>.md` if result is referenced later

### Trigger conditions

| Need | Example query |
|---|---|
| EU regulation update | "What's the latest EU ePrivacy Directive enforcement in Romania 2026?" |
| Vendor due diligence | "What are Supabase's current SOC 2 + EU data residency commitments?" |
| Competitor move | "Who launched modular home design SaaS in EU market in the last 6 months?" |
| Pricing validation | "What do comparable B2B SaaS tools charge for AI architecture tools in 2026?" |

## 📚 Tool 2: NotebookLM (Free)

**When to use:**
- Synthesize 5+ documents into a briefing
- Question a specific corpus (our 229 KB articles, legal PDFs, DPA replies)
- Generate podcast-style briefings for you to listen during commute
- Cross-reference knowledge from multiple sources

**NOT for:**
- Web search (use Perplexity)
- Documents that change often (NotebookLM re-uploads are manual)

### User setup (one-time)
1. Account at [notebooklm.google.com](https://notebooklm.google.com) (free with Google account)
2. Create notebooks for recurring themes:
   - **"ModulCA Legal"** — Privacy Policy, Terms, GDPR docs, DPA replies
   - **"ModulCA Product"** — our 229 KB articles, blog posts, Neufert refs
   - **"ModulCA Strategy"** — ECOSYSTEM_ARCHITECTURE, DASHBOARD_SPEC, META_SYSTEM
   - **"Regulations RO/NL"** — country-specific construction law + GDPR

3. Upload docs (or sync from Google Drive if you sync repo there)

### Handoff prompt format

```
▶ NOTEBOOKLM HANDOFF
Notebook: [ModulCA Legal | Product | Strategy | Regulations]
Sources to upload (if new): [list paths]
Question: [what to ask NotebookLM]
Output format: [brief summary | podcast briefing | cross-reference map]
Expected back: [key findings + direct quotes if important]
```

### User action
1. Open NotebookLM, select the right notebook
2. If new sources mentioned: upload them first
3. Paste the question
4. Read/listen to the answer
5. Copy key findings back to Claude chat

### Trigger conditions

| Situation | NBLM query |
|---|---|
| 3 DPA replies arrived, need to compare | "Compare the 3 vendor DPAs uploaded. Which have the weakest Schrems II language?" |
| Plan Q3 content | "What themes in our 229 KB articles are underrepresented?" |
| Prep investor deck | "Summarize the ModulCA ecosystem strategy in 5 bullet points suitable for a 10-min pitch." |
| Customer objection | "What do our docs say about seismic resilience of modular homes in Romania?" |

## 🎨 Tool 3: Claude Design (Included in Claude chat)

**When to use:**
- UI mockups before implementing
- Landing pages for new brands (NeufertAI, Bunkere, Treehouse)
- Pitch deck visuals, investor slides
- Brand identity exploration

**NOT for:**
- Modifying existing ModulCA UI (Claude Code does that)
- Technical architecture diagrams (Claude chat or drawio)

### User setup (one-time)
- None — access via [claude.ai/design](https://claude.ai/design) with your existing Claude account

### Handoff prompt format

```
▶ CLAUDE DESIGN HANDOFF
Artifact: [landing page | mockup | pitch slide | logo exploration]
Brand: [ModulCA | NeufertAI | Bunkere | Treehouse | CrisisReady]
Visual direction: [minimalist | organic biophilic | tech-forward | ...]
Target audience: [described]
Key elements: [hero text, CTAs, sections, must-include]
Inspiration refs: [URLs or descriptions]
Output: [React component code | image mockup | HTML]
```

### User action
1. Open Claude at claude.ai → New chat → Design mode
2. Paste the handoff prompt
3. Iterate (usually 2-4 turns)
4. Export: component code goes to `src/app/(future-brand)/page.tsx`; mockups saved to `docs/design/mockups/`
5. Summary back to Claude Code chat

## 💻 Tool 4: ChatGPT Codex CLI (€20/mo ChatGPT Plus)

**When to use:**
- Second opinion on critical code (Stripe webhook, auth, payment flow)
- Alternate approach when Claude Code is stuck
- Peer review before merging high-risk PR
- Pair programming on hard bugs

**NOT for:**
- Replacing Claude Code as primary developer
- Simple features (Claude Code handles those fine)

### User setup (one-time)
1. ChatGPT Plus subscription active
2. Install Codex CLI: `npm install -g @openai/codex` (or equivalent per current docs)
3. Authenticate: `codex auth login`
4. Test: `codex --version`

### Handoff prompt format

```
▶ CHATGPT CODEX HANDOFF
Task: [one-sentence goal]
Context files: [paths relative to repo root, Codex will read]
Current approach: [what Claude Code already tried]
Ask: [alternate approach | review this implementation | find bugs]
Success criteria: [how we know the answer is good]
```

### User action
1. Open terminal in modulca repo
2. `codex` (enters CLI)
3. Paste the handoff prompt
4. Let it read files, propose solution
5. Copy Codex's output back to Claude Code chat
6. Claude Code decides: integrate, discard, or hybrid

### Trigger conditions

| Situation | Why Codex |
|---|---|
| Stripe webhook has subtle bug | Claude Code missed it; Codex sees patterns from different training |
| Auth flow needs hardening | Double-check against OWASP before merge |
| Performance tuning | Codex often suggests different optimization angles |
| Architecture decision | Two opinions > one |

## 🔍 Tool 8: Combined Perplexity + NotebookLM (Deep Investigation)

**When to use:**
- Major strategic decisions requiring both external + internal context
- Legal/compliance investigations
- Post-incident post-mortems (what happened externally + internally)
- Annual audits / quarterly reviews

### Workflow

```
1. Claude Captain identifies question needing deep investigation
2. Captain splits it:
   - External facts → Perplexity query
   - Internal consistency → NotebookLM query against our docs
3. User runs both in parallel
4. Both results copied back to Captain
5. Captain synthesizes + writes decision doc
```

### Example: "Should we apply for Horizon Europe Cluster 3 grant?"

- Perplexity: "What was the 2026 Horizon Europe Cluster 3 success rate for first-time applicants? What are evaluator preferences?"
- NotebookLM (ModulCA Legal + Strategy notebook): "Does our current MCA + UII legal structure satisfy grant consortium requirements? What's our demonstrable crisis management expertise per our docs?"
- Captain synthesizes both → decision with rationale

## 🎯 When which tool (quick decision tree)

```
Task arrives at Captain / Ops / Specialist:

├── Is it code?
│   ├── YES → Claude Code (primary)
│   │   └── Critical path? → Also ChatGPT Codex (peer review)
│   └── NO → continue
│
├── Is it UI/visual?
│   └── YES → Claude Design
│
├── Is it research?
│   ├── Needs real-time / fresh data?
│   │   └── YES → Perplexity Pro
│   ├── Needs to query our docs/KB?
│   │   └── YES → NotebookLM
│   └── Needs social signals / reactions?
│       └── YES → Grok
│
├── Is it autonomous multi-step (background)?
│   ├── Can Claude Code + MCP do it?
│   │   └── YES → Claude Code with scheduled task
│   └── Truly needs browser+terminal+files+time?
│       └── YES → Manus AI (evaluate credits first)
│
└── Is it team coordination?
    └── YES → Claude Cowork (when team exists)
```

## 🔒 Data handling rules

- Never paste secrets / API keys / DPA replies with real vendor info into external tools unless they're in a private workspace (Perplexity Spaces, NotebookLM notebooks are private by default)
- Public-facing copy (blog drafts, pitch decks) is fine to iterate with any tool
- PII of users (emails, project data) → stays in our stack, never leaves
- Legal drafts → NotebookLM (Google) is acceptable; avoid uploading to Grok/public ChatGPT
